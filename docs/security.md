# Security and Data-Protection Practices

Version: 1.0
Document date: 2026-07-17
Governing protocol: [preregistration.md](preregistration.md)

This project handles secondary human neuroimaging data under data-use agreements. The controls below are obligations, not suggestions.

---

## 1. Secret handling

- No credential, token, API key, password, or private key is ever committed to this repository.
- No secret is pasted into a chat, an issue, a commit message, a log, or a report.
- Secrets are supplied through the local environment or an out-of-tree file, never through tracked configuration.
- No `.env` file exists in this repository, and none is needed for the current work. Repository ignore rules exclude `.env` and `.env.*` as defence in depth.
- Tracked files are scanned for credential patterns by `pre-commit` (`detect-private-key`) and by both verification scripts. These scans catch common shapes; they are a safety net, not a substitute for not writing the secret in the first place.

---

## 2. FreeSurfer license handling

fMRIPrep requires a FreeSurfer license file. It is licensed material and is treated as a secret.

- Location: `$HOME/licenses/license.txt` inside WSL2 — outside the repository entirely.
- Permissions: `600`, owner-only.
- The repository ignore rules exclude `license.txt`. This is defence in depth, not permission to place the file in the working tree.
- The license contents are never printed, echoed, read aloud, copied into a report, pasted into a chat, hashed into a public document, or committed.
- Existence and permissions are verified without reading the contents:

```bash
stat -c "%a %U %G %n" "$HOME/licenses/license.txt"
```

---

## 3. Participant data protection

- No participant data appear in logs, error messages, exception text, commit messages, or reports.
- No attempt to re-identify any participant is permitted under any circumstance.
- Raw imaging is identifiable-adjacent and never enters version control.
- Only aggregate, disclosure-safe outputs may appear in any public report or artifact. No participant-level imaging, identifier, or reconstructable feature vector is published.
- Diagnostic labels are research labels. They are not clinical diagnoses, and this system is not for clinical use.
- Data-use agreements and licenses govern all sharing. Redistribution occurs only when the governing license explicitly permits it and the permission has been verified and recorded.

---

## 4. Large-file safeguards

Imaging data are large, and a single accidental commit is effectively permanent because Git history is immutable by policy here.

Layered defences:

1. `.gitignore` excludes data directories, imaging formats, array formats, model weights, caches, and archives.
2. `.gitattributes` marks those formats binary, so no text conversion masks them.
3. `pre-commit`'s `check-added-large-files` blocks anything over 1 MB from being committed.
4. Both verification scripts fail on any file over 5 MB outside ignored directories.
5. Both verification scripts fail on any tracked path matching an imaging, derivative, or model-weight pattern.

If a large file or dataset is committed by accident, stop and report it before pushing. Do not attempt a history rewrite unilaterally — this repository's history carries the evidentiary claim that the protocol was locked before modeling, and rewriting it damages that claim.

---

## 5. Dependency supply chain

- Python packages are installed from PyPI, and GPU-enabled PyTorch from the official PyTorch index at `https://download.pytorch.org/whl/`. No other index is configured.
- Dependencies are pinned in lock files generated by a real resolution, so an install is reproducible rather than whatever happens to be current.
- `pre-commit` hook revisions are pinned. They are updated deliberately, never automatically.
- Container images are pinned by immutable digest rather than by tag, because a tag can be repointed at different content.
- No downloaded script is executed without review. Piping a remote script directly into a shell is prohibited: it executes whatever the server returns at that moment, which is neither reviewed nor reproducible.
- Installation instructions are read from the official source of the tool at the time of installation, never from a third-party blog and never copied from a stale document.

---

## 6. System boundary

This repository does not silently install or reconfigure system-wide software. WSL2 distributions, Docker Desktop, GPU drivers, R, FSL, AFNI, FreeSurfer, `.wslconfig`, global Git configuration, and global VS Code configuration are all outside its scope. Their status is discovered and reported; changing them is a deliberate manual action by the user.

---

## 7. Reporting an accidental exposure

If a secret, license, credential, or participant datum is exposed — committed, pushed, logged, or pasted — treat it as compromised. Assume disclosure; do not assume nobody noticed.

1. **Stop.** Do not push. Do not continue the task that caused it.
2. **Report it immediately** to the study lead, before attempting any remediation. A quiet cleanup destroys the evidence needed to judge the exposure's scope.
3. **Revoke and rotate** the exposed credential at its source. Rotation is what ends the exposure; removing the file does not, because the value is already out.
4. **Assess scope.** Determine whether the material reached a remote, a public repository, a log aggregator, a build service, or a third-party chat. A push to a public remote must be assumed to have been indexed.
5. **Record the incident**, including what was exposed, where it reached, when, and what was rotated. If a data-use agreement covers the exposed material, follow its notification requirements.
6. **Only then** decide on repository remediation, with the study lead. History rewriting is a joint decision, never a reflex.

For an exposed FreeSurfer license, notify the license issuer and request a replacement. For exposed participant data, the governing data-use agreement dictates the notification path, and the study lead executes it.
